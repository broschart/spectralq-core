User Manual

SpectralQ.ai is a digital investigation platform for search behavior, media dynamics, and geospatial signals. The platform overlays up to twelve independent data layers – from Google Trends to AIS vessel traffic to satellite image comparisons – on a common timeline. Only when multiple sources spike simultaneously does a hint become a finding. Analyses can be performed fully automatically via AI agents or entirely manually – or a combination of both.

Practical Analysis Workflow

Recommended sequence for a complete analysis – from setup to finished presentation.

Step 1 – Create a Project

  • Under Projects, create a new project (name, color).
  • Optionally add a project briefing: topic, research question, target audience. This text is automatically sent as context with every AI assessment.

Step 2 – Create & Assign Keywords

  • Under Keywords, create search terms – with geo, time range, and service (Web / News / YouTube …).
  • In project mode (dropdown at the top), newly created keywords are automatically assigned to the active project.
  • Assign existing keywords retroactively: select checkboxes → To Project….
  • Keywords can be assigned to multiple projects simultaneously.

Step 3 – Fetch Data

  • Manually: In the keyword list, click "Fetch All" or start individual fetches per keyword.
  • Automatically: Under Admin, set up a daily fetch schedule (time in server time).
  • The status of the last fetch is available in the Fetch Logs (Admin).

Step 4 – Open Analysis & Select Project

  • Navigate to the Analysis page.
  • In the header area on the right, select the desired project – the keyword list automatically filters to project keywords.
  • In freestyle mode (no project), all keywords are available for selection.

Step 5 – Configure Keywords & Time Range

  • Activate/deactivate keyword chips (up to 7 simultaneously).
  • Optionally select a fetch version (run tag) via the menu below the chart.
  • Restrict the visible time range via the navigator (miniature bar) or zoom directly in the chart by Ctrl+dragging.

Step 6 – Analyze the Chart

  • Crosshair: Move the mouse over the chart → tooltip with all values at the respective date.
  • Correlation: Correlation matrix at the bottom left; shift lag with ‹ / ›, open CCF chart.
  • Events appear as colored lines/areas and help to causally contextualize spikes.
  • Distance Mode (D): Span a time interval → shows ranges for different modes of transport. M opens the range map with concentric rings.
  • Decomposition: With one active keyword, activate decomposition into trend, seasonality, and noise directly in the chart (additive or multiplicative).
  • Fullscreen: Two modes available – pure chart fullscreen or layout fullscreen (controls + chart side by side). Exit with Esc.

Step 7 – Use Analysis Panels

  • Self-Similarity – detects repeating patterns in the time course of a keyword.
  • Outlier Detection – marks statistical anomalies (Z-Score / IQR).
  • Time Series Decomposition – separates trend, seasonality, and noise.
  • Related Queries & Regional Interest as complementary perspectives.

Step 7b – Lab Analyses (Test Tube Menu)

  • Spike Coincidence – detects simultaneous spikes across multiple keywords.
  • Change-Point Detection – finds structural breaks in time series (PELT algorithm).
  • Rolling Correlation – rolling Pearson correlation with phase detection.
  • Granger Causality – tests temporal predictive relationships between keywords.
  • Forecasting – prediction via Prophet or ARIMA (accessible via keyword context menu).
  • RQ Cluster Analysis – thematic clustering of related queries over time (accessible via keyword context menu).

Step 8 – Get AI Assessment

  • 🤖 AI Analysis opens an assessment of the visible data.
  • If a project is selected and contains a briefing, it is automatically sent as context.
  • Separate AI analysis buttons are also available for Self-Similarity, Outliers, Decomposition, and all lab analyses.
  • Each lab analysis automatically passes its specific computation results (changepoints, correlation phases, Granger results, etc.) to the AI.

Step 9 – Save Snapshot

  • 📸 Snapshot saves the current view (keywords, time range, zoom, correlations, statistics).
  • Assign to the active project, provide a title and comment.
  • Snapshots can be retrieved at any time and reproduced in the analysis.

Step 10 – Assemble & Export Presentation

  • Under Projects, sort snapshots via drag & drop.
  • Add text slides (title, section), map slides, project briefing slides, or other slide types via + Page.
  • Map slides: set markers with optional distance rings; comment, timestamp, and radius are editable after placement.
  • Start Presentation – fullscreen mode with keyboard navigation.
  • Export PDF – renders all slides and generates a downloadable PDF file.
💡 For recurring analyses, you can start from step 3 – project, keywords, and settings are preserved and do not need to be recreated.

Overview

SpectralQ.ai follows two parallel working principles: manual-forensic or fully automated via AI agents. Both paths lead to reproducible, hash-secured findings.

1 · Create Keywords → 2 · Fetch Data → 3 · Triangulation & Analysis → 4 · Snapshots & Report → 5 · Alerts & Monitoring
Alt. · Enter Briefing → AI Agents Investigate → Report & Project Automatically

Data Backends

  • Playwright – Controls a real browser, optionally with cookies. Most reliable method.
  • pytrends – Lightweight Python library without a browser. More susceptible to rate limiting.
  • SerpAPI – External API that reliably delivers Google Trends data (100 free requests/month). Requires an API key.

During fetching, SpectralQ.ai tries the backends sequentially (fallback principle). Order and activation are configured in the Admin section.

Keywords /

The start page is the central management hub for all monitored search terms. Keywords are created, configured, and data is fetched here.

Create Keyword

  • Search Term – The term as it would be entered in Google Trends.
  • Geo – Country or region code (e.g., DE, US-CA). Leave empty for worldwide data.
  • Time Range – Preset time windows: last hour, 4 hours, 24 hours, 7 days, etc. Determines data granularity (hourly vs. daily).
  • Service – Data category: Web Search, News, Images, YouTube, or Shopping.
  • Note – Optional internal free-text field.
  • Active/Inactive – Inactive keywords are skipped during automatic fetching.

Translator

  • The Translator button (🌐) automatically translates a term into EN, ES, FR, ZH, and RU and suggests matching countries.
  • Clicking a translation chip applies the term and country directly to the form.

Project Assignment

  • Project Mode: Select a project in the dropdown at the top – the list filters to assigned keywords. Newly created keywords are automatically assigned to this project.
  • Freestyle Mode: No project selected – all keywords visible, no automatic assignment.
  • To Project…: Select checkboxes → "To Project…" – opens a modal for assigning or removing from any projects. Keywords can belong to multiple projects simultaneously.
  • Assigned projects are displayed as colored badges in the keyword row.

Table & Actions

  • Multi-select (up to 7 keywords): Select checkboxes, then "Open Analysis" – launches the analysis page with pre-selected keywords.
  • Fetch Now – immediately triggers a data fetch for this keyword only.
  • Related Queries – opens a modal with rising and top related search terms (saved during the last fetch).
  • Regional Interest – opens a modal with a bar chart; switchable between state/province level and city level.
  • Edit – modifies all keyword parameters retroactively.
  • Delete – permanently removes the keyword and all associated trend data.
💡 Using search and the "Active / Inactive" filter, even large keyword lists can be quickly browsed. The project selection is synchronized across pages with the analysis.

Analysis /analysis

The core of the application: interactive time series visualization with comparison, correlation analysis, temporal statistics, and regional evaluation.

Project & Keyword Selection

  • In the header area on the right, a project can be selected – the keyword list automatically filters to keywords assigned to the project.
  • In freestyle mode (no project), all keywords are available.
  • The project selection is synchronized with the Keywords page (localStorage).
  • The page without a selected keyword shows a search field and the most recently used keywords as quick chips.
  • Up to 7 keywords can be selected simultaneously. Each receives its own color.
  • URL parameter ?kw=id1,id2 loads keywords directly on page load.

Chart Interaction

  • Crosshair Cursor – When moving the mouse, a vertical line is displayed and all values at the cursor date are shown as a tooltip.
  • Shift + Drag – Marks a time range (blue) as a time range event.
  • Drag – Zoom box for direct zoom into an area.
  • Navigator – Narrow overview bar below the main chart. The visible section can be shifted by dragging.
  • Fetch Version – Multiple fetches of the same keyword (different run tags) can be displayed as separate curves.

Triangulation (External Data Overlay)

The purple "Triangulation" button (to the right of AI Analysis in the chart toolbar) opens a centered panel with all external data sources. These are displayed as additional curves in the timeline graph to reveal correlations and anomalies. During loading, a pulsing colored indicator above the chart shows progress.

The sources are organized into three groups:

Text & Media

  • Wikipedia Pageviews/Edits – Daily view counts and edits of the associated Wikipedia articles. Separate right Y-axis (orange). The article title is resolved automatically. Clicking the article title opens the author and edit check with revision analysis and reputation assessment.
  • GDELT Media Reports – Daily article count worldwide from the GDELT project. Separate Y-axis (light blue). Shows media intensity of a topic.
  • Bluesky Posts – Posting volume on Bluesky for freely selectable keywords. Separate Y-axis (blue). Only appears if Bluesky credentials are configured in Admin (Handle + App Password).
  • Telegram Messages – Message volume in public Telegram channels. Separate Y-axis (teal). Only appears if Telegram API credentials are configured in Admin and a session is authenticated.

Financial Markets

  • Yahoo Finance – Stock prices and/or trading volume for ticker symbols. Enter a company name or symbol in the search field – an autocomplete search suggests matching tickers (e.g., TSLA, SAP.DE, BTC-USD). Select by clicking, remove via × button or Backspace. Up to 5 tickers simultaneously. Dropdown selection: price, volume, or both. No API key required. Separate Y-axes for price (green) and volume.

Geospatial Sources

Geospatial sources are based on Watch Zones – geographic monitoring regions drawn directly on the map as polygons or rectangles. Each Watch Zone has a type (satellite, weather, seismic, nightlights, etc.) and a geometry. If no matching Watch Zone exists, the checkbox is grayed out and the hint "Create Watch Zone first" appears. If zones exist, the first one is automatically preselected.

  • Sentinel-2 NDVI – Vegetation index from Sentinel-2 satellite imagery. Detects land changes (deforestation, construction, fires). Requires Copernicus credentials in Admin. Mark region via map picker.
  • DWD / NOAA – Weather and gauge data. Four measurement types selectable: water levels, precipitation, weather warnings, storm tracks. Bar chart for precipitation/water levels, line for warnings/storms. Separate Y-axis (blue). Requires Watch Zone of type "weather".
  • Satellite Observation – Marks available Sentinel-2 acquisition dates as vertical lines in the chart. Clicking a marker opens the satellite image with before/after comparison and NDVI analysis. Requires Watch Zone of type "satellite" and Copernicus credentials.
  • Seismics (USGS) – Earthquake data from the US Geological Survey. Shows maximum magnitude per time slot as red dots (size proportional to strength). Separate Y-axis (red). Requires Watch Zone of type "seismic". No API key needed.
  • Nighttime Lights (NASA VIIRS) – Mean brightness from NASA VIIRS nighttime light imagery. Detects changes in lighting and infrastructure (power outages, construction activity, conflict zones). Separate Y-axis (yellow). Requires Watch Zone of type "nightlights". No API key needed, uses NASA GIBS WMS.

General Notes

  • Text/media sources are displayed as dashed lines, geospatial sources as solid lines with their own Y-axis.
  • All data is cached – toggling on/off does not require a new API call.
  • Wikipedia, GDELT, Yahoo Finance, Seismics, and Nighttime Lights are available without an API key. Bluesky, Telegram, NDVI, and Satellite require credentials.
  • The Triangulation panel closes by clicking outside, the × button, or Esc.

Decomposition (Time Series Decomposition in Chart)

  • With exactly one active keyword, a Decomposition checkbox with algorithm selection appears to the left of "Normalize".
  • Two models are available: Additive (Y = T + S + R) and Multiplicative (Y = T × S × R).
  • Activation overlays three additional lines in the main chart: Trend (yellow), Seasonal (green), and Residual (purple).
  • Seasonality and noise are displayed on a separate right Y-axis.
  • Hover values are rounded to 2 decimal places in decomposition mode.
  • When decomposition mode is active, the decomposition data is automatically passed to the AI Analysis.
  • When a second keyword is activated, decomposition is automatically deactivated.

Fullscreen Modes & Magnifier

  • Chart Fullscreen (expand icon) – displays only the chart in fullscreen, including navigator. Optimized for large monitors and presentations.
  • Layout Fullscreen (columns icon) – shows the control area (search, keywords, fetch runs, display area) on the left (25%) and the chart on the right (75%) side by side in fullscreen. Ideal for collaborative work and team presentations.
  • Magnifier – Precisely enlarges any chart area: move the cursor over the area and activate the magnifier. Especially useful for detailed analysis of narrow time windows or during team presentations on large displays.
  • All modes are exited with Esc and are mutually exclusive.

Index Mode

  • Key I – activates Index Mode. All curves are normalized to the value at the cursor date (= 100), making relative changes directly comparable.
  • Moving the mouse shifts the reference point in real time.
  • Key F – fixes the current reference point. The normalization stays frozen even when the mouse moves further. Pressing F again releases the fix.
  • A blue badge "Index Mode · F = fix" or "Index fixed" appears at the top of the chart.
  • Pressing I again or Escape exits index mode and restores the original values.

Distance Mode

  • Key D (mouse must be over the chart) – activates Distance Mode. A start point is set; moving the mouse spans a time interval. A semi-transparent arrow visualizes the span.
  • The tooltip shows how far one could travel in that time with different modes of transportation (car, bicycle, on foot).
  • Pressing D again – fixes the arrow and values. Another D or Escape exits distance mode.
  • Key M or clicking on the chart in distance mode – opens the Map Mode (interactive map with concentric rings for each mode of transport).
  • In map mode: address search or 📍 geolocation sets the origin point. Clicking a transport mode button in the legend zooms to the corresponding ring.
  • Map Snapshot – saves the current map view as a map slide in a project (select project in the dropdown next to the save button).

Correlations

  • For each keyword pair, the Pearson correlation coefficient r is automatically calculated.
  • Color coding: strongly negative (red) → no correlation (gray) → strongly positive (green).
  • Time Shift (Lag): With ‹ / › (±1 step) and « / » (±7 steps), curve B can be shifted in time against curve A. The correlation updates live; the shifted curve is displayed as dashed.
  • The value in the center shows the current shift in hours (h) or days (d). An ✕ resets to lag 0.
  • CCF (Cross-Correlation Function): The CCF button opens a layer with a bar chart of all correlation values for lags from −20 to +20. Clicking a bar applies that lag directly to the manual shift.
💡 A positive lag means: B lags behind A. A negative lag means: B leads A. The curve shifts in the chart accordingly to the left (−) or right (+).

Temporal Statistics

  • Shows average trend values by day of week and time of day (only for hourly data).
  • Provides insight into recurring patterns in search interest (e.g., peaks on Mondays or in the evening).

AI Analysis

  • 🤖 AI Analysis – sends the visible time series data to the configured language model and delivers a structured assessment.
  • If a project with a project briefing is selected in the header, it is automatically sent as context – the AI can better contextualize the topic.
  • Separate AI buttons are also available in the Self-Similarity, Outlier Detection, Time Series Decomposition, and all lab analysis panels.
  • Each analysis passes its computed data (changepoints, correlation phases, Granger results, forecasts, etc.) automatically to the AI.
  • AI provider and model are configured under Admin.
  • Non-admin users automatically use the superadmin's API keys, unless their own API has been authorized.

Additional Panels

  • Related Queries – shows rising and top terms related to the selected keyword.
  • Regional Interest – bar chart at state/province or city level.
  • Daily & Weekly Statistics – average values visualized by time unit.
  • Self-Similarity – detects repeating time windows in the course of a keyword.
  • Outlier Detection – marks statistical anomalies via Z-Score or IQR method.
  • Time Series Decomposition – separates trend, seasonality, and noise (additive or multiplicative). Alternatively usable directly in the main chart via the Decomposition checkbox.

Lab Analyses Test Tube Menu

Advanced statistical methods, accessible via the test tube dropdown in the toolbar or via the context menu of individual keywords.

Spike Coincidence

  • Detects simultaneous spikes across multiple active keywords.
  • Coincidences are highlighted as vertical markers in the chart.
  • The results (coincident data points, involved keywords) are automatically passed to the AI.

Change-Point Detection

  • Finds structural breaks in time series using the PELT algorithm (Pruned Exact Linear Time).
  • The chart shows detected changepoints as vertical lines and segments with alternating backgrounds.
  • Segment statistics (mean, standard deviation, rate of change) are calculated and passed to the AI.
  • Accessible via the lab dropdown or the keyword context menu.

Rolling Correlation

  • Calculates the rolling Pearson correlation between keyword pairs over a configurable time window.
  • The chart shows the correlation trajectory (Y-axis −1 to +1) with color-coded zones (positive/negative/neutral).
  • Phase detection identifies contiguous time periods of consistent correlation direction.
  • All pairs or a single pair can be selected.
  • Phases and average correlation are passed to the AI.

Periodicity Filter (FFT)

  • Detects recurring cycles in a time series using Fast Fourier Transform (FFT).
  • Shows the dominant frequencies and their period lengths (e.g., 7-day rhythm, 30-day cycle).
  • The results are automatically passed to the AI for interpretation.

Granger Causality

  • Tests temporal predictive relationships between all keyword pairs in both directions.
  • Results table with direction, optimal lag, F-statistic, p-value, and significance rating (★/★★/★★★).
  • Expandable lag details show all tested lags per pair.
  • Network hub summary: which keyword Granger-causes the most others.
  • Important: Granger causality ≠ true causality – it measures predictive power, not causal mechanisms.

Forecasting

  • Calculates a time series forecast for a single keyword.
  • Two models: Prophet (automatic seasonality and changepoint detection) or ARIMA (auto-configured order).
  • Configurable forecast horizon (number of steps).
  • The chart shows historical data, forecast line (green dashed), and confidence band.
  • Forecast results are passed to the AI for interpretation.
  • Accessible via the keyword context menu (not in the lab dropdown).

RQ Cluster Analysis

  • Analyzes related queries of a keyword across all fetch time points.
  • The AI groups all queries into thematic clusters and evaluates their temporal development.
  • Identifies emerging, stable, and disappearing clusters as well as breakout queries.
  • Includes chat function for follow-up questions about the cluster analysis.
  • Accessible via the keyword context menu (not in the lab dropdown).
💡 All lab analyses automatically pass their computed data to the AI assessment. The AI receives a specialized prompt with an explanation task and the specific analysis results in each case.

AI Project Assistance /start

Fully autonomous analysis with a 4-agent team: A search term or briefing is all it takes – the AI independently creates a project, fetches data, creates snapshots, and delivers a forensically robust report with confidence analysis and counter-thesis.

Input

  • Search Term – The seed keyword from which the research starts. Automatically checked for language and appropriate geo context.
  • Briefing – Optional free-text field for the research question or context. The briefing significantly steers the research strategy, e.g., it can influence the geo focus (question about Russia → search in RU with Russian keywords).
  • Intensity – Slider (1–5). Higher values lead to more iterations, more keywords, and more intermediate snapshots.

4-Agent Architecture

APA operates with four specialized AI agents that collaborate in each iteration:

  • Coordinator – Thinks strategically and is hypothesis-driven. Formulates an explicit hypothesis in each round, commissions its testing, and actively searches for counter-evidence. Recognizes that many questions can only be measured indirectly. Uses cross-project memory to incorporate findings from earlier analyses.
  • Trend Researcher – Specialist for trend data. Analyzes time series for peaks, seasonal patterns, correlations, and anomalies. Reports to the coordinator with specific data values and time periods.
  • RQ Researcher – Specialist for related queries. Identifies topic clusters, semantic connections, and rising trends. Suggests new, often lateral keywords to the coordinator.
  • Critic – Devil's advocate. Activated after the iterations are complete and systematically questions all conclusions: Are correlations being interpreted as causation? Are there alternative explanations? Was only confirming data sought? Their feedback is incorporated into the final report.

Trend and RQ Researcher work in parallel. The Critic works adversarially – they are the only agent that does not act cooperatively.

Process

  • After starting, a log monitor opens showing progress in real time (SSE-based). Agent names appear in purple/bold.
  • Phase 1 – Seed Fetch & Memory: The seed keyword is fetched from Google Trends. The system checks whether overlapping keywords have already been analyzed in previous projects (cross-project memory).
  • Phase 2 – Automatic News Scan: Immediately after the seed fetch, up to 15 current news articles about the seed keyword are automatically searched via Google News and their full texts scraped (with cleanup of cookie banners and boilerplate). The articles are created as events in the project and the contents are provided to the coordinator as context. This gives the AI access to the current news landscape on the topic from the very beginning.
  • Phase 3 – Hypothesis-Driven Iteration: In each iteration, the coordinator formulates an explicit hypothesis, commissions targeted fetches and analyses to test it, and evaluates after the researcher reports: confirmed, refuted, or open? The research strategy actively seeks counter-evidence. The coordinator has access to the scraped news content and can use it for hypothesis formation.
  • Phase 4 – Critic Review: After the iterations are complete, the critic agent analyzes all conclusions adversarially: logical errors, confirmation bias, alternative explanations, missing counter-evidence, over-interpretations.
  • Phase 5 – Forensic Final Report: The report contains confidence values for each core statement, a mandatory section "What this analysis does not prove," historical pattern comparison, the critic's feedback, and the findings from the news scan.
  • When data is missing for a specific country, an automatic geo fallback to worldwide data is attempted.

Statistical Analyses & Events

  • The coordinator can run statistical analyses on the fetched data – e.g., change-point detection, rolling correlation, spike coincidence, periodicity, outlier detection, time series decomposition, self-similarity, Auto Correlate (automatically finds the most strongly correlated keywords including optimal lag), and Forecasting (time series prediction via Prophet or ARIMA).
  • The results feed directly into the assessment and the final report.
  • Stored events (manually created annotations) are automatically considered as context – the AI can thus correlate spikes with known events.

External Data Sources

  • Automatic News Scan – Performed immediately after start: Up to 15 news articles about the seed keyword are found, scraped, and created as events. The coordinator can additionally trigger targeted news scans with different search terms or time ranges.
  • Event Probe – Forensic short-window analysis (1 or 7 days) around a specific event date. The system pulls Google Trends data for a narrow time window and automatically analyzes:
    • Foreknowledge Indicator – Significant search volume BEFORE the event → possible indication of foreknowledge or planning
    • Reactive Pattern – Search volume only AFTER the event → normal reaction pattern
    • Suspicious – Search volume ONLY BEFORE the event, not after → particularly worth investigating
  • Wikipedia Pageviews – Daily view counts of Wikipedia articles as an independent signal alongside Google Trends. Serves as cross-validation: Do search trends align with information interest on Wikipedia? Provides absolute numbers (in contrast to Google Trends' relative 0–100 scale). Spikes frequently correlate with news cycles.
  • Wikipedia Edits – Edit history of Wikipedia articles: Who edits, when, how often? Detects edit spikes (PR campaigns, crisis management), analyzes author reputation and anonymous vs. registered edits. Forensically relevant for detecting whitewashing or coordinated article manipulation.
  • GDELT Media Volume – Daily article count from the GDELT project, which monitors worldwide news media. Forensically relevant: Was there first searching or first reporting? Missing media coverage with high search volume is suspicious. Enables the distinction between organic and media-driven search interest.
  • Yahoo Finance – Historical price data, volume, and volatility for stocks, ETFs, cryptocurrencies, and indices. Forensically relevant: Correlation between search interest and market movements, insider hints (was there first searching, then the price moved?), volume spikes during news events.
  • NDVI Vegetation Analysis – Sentinel-2 satellite images provide the vegetation index (NDVI) for any region. Detects land changes: deforestation, construction projects, natural disasters, fires, renaturation. Requires Copernicus credentials.
  • ACLED Conflicts – Armed conflicts and political violence as bar chart (events per day). Separate Y-axis (red). Requires Watch Zone of type "acled" and ACLED account.
  • Air Quality (OpenAQ) – PM2.5 daily averages from monitoring stations in the zone. Separate Y-axis (green). Requires Watch Zone of type "airquality" and OpenAQ API key.
  • Power Grid (ENTSO-E) – Daily average load of the power grid. Separate Y-axis (yellow). Requires Watch Zone of type "powergrid" and ENTSO-E token.
  • Migration (UNHCR) – Refugee and asylum numbers per year. Dashed reference line for short time ranges, trend curve for multi-year views. Separate Y-axis (violet). Requires Watch Zone of type "migration". No API key needed.
  • Telegram Monitor – Mention volume in public Telegram channels per day. Dashed line (teal). Requires Watch Zone of type "telegram_monitor" with configured keywords.
  • Bluesky Monitor – Post volume on Bluesky per day. Separate Y-axis (blue). Requires Watch Zone of type "bluesky_monitor" with configured keywords.
  • OSM Changes – Number of OSM edits per day as green bars. Separate Y-axis. Requires Watch Zone of type "osm_changes". No API key needed.
  • Wayback Frequency – Snapshots/day of a URL as cyan bars. Peaks indicate media attention. Separate Y-axis. Requires Watch Zone of type "wayback_cdx". No API key needed.
  • All external sources are used autonomously by the coordinator – e.g., for triangulation of unexplained peaks or when the briefing suggests a forensic research question.
  • The triangulation menu includes a search field for quickly filtering available sources. Only plugins with stored credentials are shown.

Intelligent Search Strategy

  • Indirect Measurement: Questions like "Who is more popular?" cannot be measured directly. The coordinator recognizes this and instead compares the individual entities (e.g., party names, politician names).
  • Geo Adaptation: The briefing can influence the geo context. A question about Russia leads to searches in geo=RU with Russian keywords.
  • Search Behavior: Keywords are formulated as people actually search – short, without filler words, in the language of the target geo.
  • Duplicate Protection: Identical snapshot charts are automatically prevented. Each snapshot shows a targeted selection of 1–3 keywords.

Result

  • A new project is automatically created, bundling all keywords, snapshots, and the report.
  • The report appears in a report overlay and is saved as a slide in the project.
  • All snapshots contain Markdown-formatted comments with analytical assessments.
  • The report contains a confidence analysis: Each core statement is rated with a confidence value (High/Medium/Low) and justification (data density, time period, converging signals).
  • Mandatory section "What this analysis does not prove": Explicitly states which conclusions the data does not support, which alternative explanations exist, and which data gaps remain.
  • Historical Pattern Comparison: Automatic comparison with earlier surveys – detects pattern repetitions, trend reversals, and level changes.
💡 The AI Project Assistance is particularly suited for getting started with a new topic – it quickly provides a data-based overview before a deeper manual analysis follows. Use the briefing field to specify the research question – the more specific the briefing, the more targeted the research.

Events /events

Events are annotations in the trend chart – created manually or automatically detected from news analyses. The page combines event management, news research, and AI-powered news briefing.

Manage Events

  • Point Event – appears as a vertical line in the chart at a specific date.
  • Range Event – appears as a colored area between two dates.
  • Title & Description – are displayed in the event's tooltip.
  • Color – freely selectable via color picker; determines line or area color in the chart.
  • Events are displayed on all analysis charts and can be edited or deleted at any time.
  • Stored events are automatically passed to the AI Analysis and the AI Project Assistance as context.

Find News

  • Search Term – Searches for current news articles via Google News RSS.
  • Date Range – Optional from/to filter in date format (DD.MM.YYYY).
  • Found articles are displayed as cards with title, source, date, and link.
  • Word Cloud – Shows the 20 most frequent terms from the search result headlines. Clicking a term filters the result list (OR combination for multiple active filters). Clicking again removes the filter.

News Briefing

  • Analyze News – Scrapes the full texts of all found articles and sends them to the configured language model for analysis.
  • The result appears in a two-column modal: briefing on the left, sidebar with actions on the right.

Detected Events (Table)

  • Directly below the summary, a table shows all events detected by the LLM with date, time, message, source count, and location.
  • The LLM extracts the actual time of the occurrence from the article texts – not the publication date.
  • Each row has a checkbox. The "All" checkbox in the header selects all events at once.
  • Import as Events – Saves the selected rows as point events in the database (with purple color), assigned to the active project.

Story Presentation

  • Present as Story – Opens a fullscreen map view that navigates chronologically through all detected events.
  • Each event is displayed with date, time, description, sources, and location on an interactive map (CartoDB Voyager).
  • For multiple locations per event (e.g., "USA, Iran"), separate maps are shown side by side, each centered on the respective location.
  • The navigation buttons (left/right) show the time difference to the next event (e.g., "2 days ago", "in 3h 15m", "simultaneous") as well as the geographic distance in kilometers (straight line).
  • All event locations are geocoded in advance (Nominatim) and displayed as subtle dots on the map; the current location is highlighted.

Watch Zones /events → Live View

Monitoring zones for real-time, satellite, and website data. Each zone has a type and is opened via the map icon in the events view.

Zone Types

Geospatial Monitoring

Alerts /alerts

Automatic notifications when relevant changes occur in the trend data. Triggered events appear on the dashboard.

Alert Types

  • Appearance – triggered when a keyword appears in the related queries that was not previously present.
  • Disappearance – triggered when a keyword disappears from the related queries.
  • Spike – triggered when the current trend value exceeds a threshold. Threshold definable as absolute value (0–100) or as percentage increase compared to the average.
  • Volume Appears – Total volume in the defined time window increases significantly.
  • Volume Disappears – Total volume in the defined time window drops significantly.

Configuration

  • Keyword Scope: "All Keywords" or a manual selection of specific keywords.
  • Threshold: For spike alerts, selectable as absolute value or percentage value.
  • Time Window: For volume alerts, the window specifies over how many data points the average is calculated.
  • Alerts can be individually activated and deactivated.
💡 Alerts are evaluated with every automatic fetch. Via "Check Now" on the dashboard, the evaluation can be triggered manually.

Audit Trail /audit

Hash-chained log of all project-relevant actions. Enables forensic traceability and integrity verification of the entire analysis chain.

Activation

  • The Audit Trail is activated/deactivated in the Admin section via a toggle.
  • When activated, Audit Trail appears as its own menu item to the left of Statistics.
  • When the Audit Trail is deactivated, no entries are written and the menu item is hidden.

Logged Actions

  • snapshot_create – Snapshot created (manually or via APA)
  • snapshot_delete – Snapshot deleted
  • analysis_run – AI analysis performed (with analysis mode and keywords)
  • keyword_fetch – Manual data fetch started (bulk or individual)
  • project_create / project_delete – Project created or deleted
  • apa_start / apa_complete – APA analysis started or completed
  • export_docx / export_pdf – Export as Word or PDF document

Hash Chaining

  • Each audit entry contains a SHA-256 hash computed from the entry data and the hash of the preceding entry.
  • This chaining forms a mini blockchain: Retroactive insertion, deletion, or modification of entries is detectable.
  • The first entry uses GENESIS as the predecessor hash.

Integrity Verification

  • Via the Verify Hash Chain button, the entire chain is verified.
  • Result: Green message if the chain is intact, red message if manipulation is detected.

Filter and Navigation

  • Project Filter – Narrow entries by project.
  • Action Filter – Filter entries by action type (e.g., only exports or only snapshots).
  • Paginated navigation with 50 entries per page.

Forensic Data Integrity

  • Snapshots receive a SHA-256 content hash upon creation that proves the data has not been altered.
  • The content hash is recorded in the audit trail and can be individually verified via /api/snapshots/<id>/verify.
  • In DOCX exports, the SHA-256 hash of the snapshot is displayed.

Developer Documentation

Technical reference for the plugin system, API architecture, and platform extension. This section is intended for developers who want to create their own plugins or extend existing ones.

Plugin Architecture

  • All plugins inherit from BasePlugin (plugins/base.py). Type-specific classes (WatchZonePlugin, AIPlugin) extend this base.
  • The PluginManager (plugins/__init__.py) manages the central registry. Plugins are automatically discovered and registered on app start via discover_all().
  • Each plugin defines plugin_type (e.g., "watchzone", "ai") and plugin_id (e.g., "vessel", "trend_analysis").

File Structure (Bundled Plugins)

  • plugins/base.py – BasePlugin class with meta dict and is_available() check.
  • plugins/__init__.py – PluginManager (registry, type handlers, enable/disable, discovery, bundle support).
  • Each plugin is its own folder with __init__.py, optionally i18n.json, templates/, and static/: 
    plugins/watchzone/vessel/
├── __init__.py          # Plugin class
├── i18n.json            # Translations (de/en/fr/es)
├── templates/_panel.html # Jinja2 fragment
└── static/vessel.js     # JavaScript
  • plugins/watchzone/ – Watch zone plugins (geo, web, search).
  • plugins/ai/ – AI plugins (trend analysis, scientific paper, project assistant, etc.).
  • plugins/analysis/ – Analysis plugins (forecast, outlier, CPD, Granger, etc.).

Plugin Meta

  • Each plugin defines a meta dict with: label, icon_svg, color, description, required_credentials, i18n.
  • required_credentials – List of required AppSetting keys (e.g., ["censys_api_id", "censys_api_secret"]). The generic is_available() method automatically checks User → Global → Env fallback.
  • i18n – Translations in 4 languages (de, en, fr, es), which are automatically merged into the frontend i18n system.

Create a New Watch Zone Plugin

  • Create a new folder under plugins/watchzone/ (e.g., plugins/watchzone/my_sensor/).
  • __init__.py: Inherit from WatchZonePlugin, set plugin_id and meta.
  • Override methods: live_handler() for live data, history_routes() for history API, ai_tools() + ai_tool_handler() for AI integration, analysis_provider() for triangulation.
  • Optional: i18n.json (translations), templates/ (HTML fragments), static/ (JS/CSS).
  • At the end of __init__.py: PluginManager.register(MySensorPlugin()).
  • The plugin is automatically discovered and registered on the next app start. i18n.json, templates, and static files are automatically included.

Create a New AI Plugin

  • Create a new folder under plugins/ai/.
  • Inherit from AIPlugin. Default: required_credentials: ["anthropic_api_key"].
  • The AIPlugin base class provides is_available() with provider fallback (checks active LLM provider of the user).
  • API routes are defined via api_routes() and automatically registered on app start.
  • LLM call via call_llm() from plugins/ai/_llm.py — supports Anthropic, OpenAI, Gemini, Mistral automatically.
  • Quota check and guard via _guard().

API Permission System

  • Core APIs (SerpAPI, NewsAPI, LLM providers) are mapped to groups via _KEY_TO_API_GROUP. Plugin-specific credentials (Bluesky, Telegram, Copernicus, Censys, AISHub, etc.) are managed via the plugin credential system.
  • Per user and API group, there are three permission levels: admin_key (use superadmin keys), own_key (own key), disabled (locked).
  • _resolve_api_key(key, user_id) resolves automatically: user key → admin key → env variable, depending on the policy.

Plugin Access Control (Enterprise Only)

  • Only in multi-user mode. The superadmin controls access per user and plugin via user management (/users).
  • Three levels per plugin: Superadmin API (admin keys), Own API (user keys), Disabled (locked).
  • Stored as plugin_access_<type>_<id> in AppSetting. Missing entry = allowed (default).
  • Locked plugins appear on the plugin page as "Locked by Administrator" and cannot be activated or configured.

LLM Quota & Own Key

  • Users with own_key permission for the active LLM provider are not subject to any quota.
  • In this case, quota displays (statistics page, navigation bar) are automatically hidden.
  • _user_uses_own_llm_key(user_id) checks the active provider and its API permission.

REST API Endpoints (Plugins)

  • GET /api/plugins – All plugins with status, credentials, API groups per user.
  • PUT /api/plugins/<type>/<id>/toggle – Activate/deactivate plugin.
  • PUT /api/plugins/<type>/<id>/credentials – Save credentials.
  • PUT /api/plugins/ai-settings – Set AI provider and model.
💡 New plugins are automatically discovered on app start – no manual registration in app.py needed. Create a folder under plugins/<type>/, inherit from the appropriate base class, and call PluginManager.register().

Open-Core Architecture

  • The system supports two operating modes via MULTI_USER in the .env file:
  • Community (MULTI_USER=false) – No login, single user, auto-login as phantom user. Admin, audit, and user management are omitted.
  • Enterprise (MULTI_USER=true) – Multi-user with login, roles, quotas, and user management.
  • The auth abstraction resides in auth.py: In community mode, login_required and superadmin_required are no-ops.
  • User management (login, user CRUD, API policies) resides isolated in enterprise.py — only loaded in enterprise mode.
  • For GitHub open-core: omit enterprise.py, templates/users.html, and templates/login.html.
